## ✨ الميزات الرئيسية
- **🚀 أداء عالي:** يعمل البوت بكفاءة وسرعة بفضل بيئة Node.js.
- **🐐 توافق مع GoatBot:** تم تصميم البوت ليكون متوافقًا مع أوامر ومشاريع GoatBot القديمة، مما يسهل عملية النقل والتحديث.
- **🛠️ أدوات إدارة متكاملة:** أوامر قوية للمشرفين مثل الحظر (`ban`)، التحذير (`warn`)، وإعادة التشغيل (`restart`).
- **⚙️ قابلية للتخصيص:** سهولة تعديل البادئة، المشرفين، والإعدادات الأخرى عبر ملفات `config.json` و `configCommands.json`.
- **🎉 أوامر ترفيهية:** مجموعة من الأوامر الممتعة للتفاعل وإضافة جو من المرح.
- **📞 نظام تواصل:** يمكن للأعضاء التواصل مع الإدارة عبر أمر `callad` بسرية تامة.
- **🔒 حماية الكود:** تم تشفير الملفات الأساسية لحماية حقوق المطور.

## 📖 خطوات التنصيب
1.  **تحميل المشروع:** قم بتحميل المشروع أو استنسخه عبر `git clone`.
2.  **تثبيت الحزم:** افتح الطرفية (Terminal) في مجلد المشروع ونفّذ الأمر التالي:
    ```bash
    npm install
    ```
3.  **إعداد الحساب:** ضع `appstate` الخاص بحساب البوت في ملف `account.txt`.
4.  **ضبط الإعدادات:** راجع قسم **"دليل ملء ملف `config.json`"** أدناه.
5.  **تشغيل البوت:** لبدء تشغيل البوت، استخدم الأمر التالي:
    ```bash
    node index.js
    ```

## ⚙️ دليل ملء ملف `config.json`
ملف `config.json` هو قلب البوت. إليك شرح لأهم الحقول:

```json
{
  "prefix": ".",
  "admins": ["1000xxxxxxxx"],
  "botName": "NeroBot",
  "database": {
    "useMongoDB": false,
    "mongoURI": ""
  },
  "autoRestart": {
    "time": 0
  }
}
```

- **`prefix`**: العلامة التي تسبق الأوامر (مثلاً: `.` أو `!`).
- **`admins`**: قائمة بمعرفات حسابات الفيسبوك (UID) الخاصة بالمشرفين. يجب أن تكون داخل علامتي تنصيص `""`.
  - **مثال صحيح:** `["1000123456789", "1000987654321"]`
- **`botName`**: اسم البوت الذي سيظهر في بعض الرسائل.
- **`database`**: إعدادات قاعدة البيانات. إذا كنت لا تستخدم MongoDB، اترك `useMongoDB` على `false`.
- **`autoRestart`**: لإعادة تشغيل البوت تلقائيًا بعد فترة زمنية محددة (بالمللي ثانية). اتركها `0` لإيقافها.

## 🧩 دليل المطور: صنع أمر جديد
لإضافة أمر جديد، اتبع الخطوات التالية:

1.  اذهب إلى مجلد `SYSTEM/cmds`.
2.  أنشئ ملفًا جديدًا باسم الأمر (مثال: `welcome.js`).
3.  استخدم القالب التالي كأساس لأمرك:

```javascript
module.exports = {
    config: {
        name: "ترحيب", // اسم الأمر الأساسي
        aliases: ["welcome", "مرحبا"], // أسماء بديلة للأمر
        version: "1.0",
        author: "اسمك",
        countDown: 5, // فترة الانتظار بين كل استخدام
        role: 0, // 0 للجميع، 1 للمشرفين، 2 للمطور
        shortDescription: { ar: "رسالة ترحيب" },
        longDescription: { ar: "يرسل رسالة ترحيب للمستخدم" },
        category: "عام", // فئة الأمر
        guide: { ar: "{pn}" } // كيفية استخدام الأمر
    },

    // هذه هي الدالة الرئيسية التي سيتم تنفيذها عند استدعاء الأمر
    onCall: async function({ api, event, args }) {
        // الكود الخاص بك هنا
        const userName = (await api.getUserInfo(event.senderID))[event.senderID].name;
        return api.sendMessage(`أهلاً بك يا ${userName}!`, event.threadID, event.messageID);
    }
};
```

4.  **شرح:**
    - **`config`**: يحتوي على جميع معلومات وبيانات الأمر.
    - **`onCall`**: هي الدالة التي تحتوي على منطق الأمر. يتم تمرير كائنات مثل `api` و `event` إليها للتعامل مع الرسائل والإجراءات.

## 🐐 التوافق مع GoatBot
هذا البوت ليس مجرد بوت جديد، بل هو جسر للمطورين الذين اعتادوا على بنية **GoatBot**. تم بناء `NeroBot` ليكون متوافقًا تمامًا مع الأوامر القديمة، حيث يعمل `GoatBot` كاسم مستعار (Alias) لـ `NeroBot`. هذا يعني أنه يمكنك تشغيل ملفات الأوامر القديمة دون الحاجة إلى تعديلها.

## 🐞 الأخطاء الشائعة وكيفية حلها

- **خطأ في تسجيل الدخول / `Login approval needed`**
  - **السبب:** انتهت صلاحية ملف `appstate` أو تم تسجيل الخروج من الحساب.
  - **الحل:** قم باستخراج `appstate` جديد من متصفحك وضعه في ملف `account.txt`.

- **`"حدث خطأ أثناء onCall في الأمر..."`**
  - **السبب:** يوجد خطأ برمجي (Bug) داخل ملف الأمر نفسه.
  - **الحل:** افتح الطرفية (Console) واقرأ تفاصيل الخطأ التي تظهر باللون الأحمر. عادة ما يخبرك الخطأ بالسطر الذي توجد فيه المشكلة في ملف الأمر.

- **`"لا يمكن العثور على الأمر..."`**
  - **السبب:** إما أنك كتبت اسم الأمر بشكل خاطئ، أو أن الملف الخاص بالأمر يحتوي على خطأ كبير منعه من التحميل عند بدء تشغيل البوت.
  - **الحل:** تأكد من كتابة الأمر بشكل صحيح. إذا استمرت المشكلة، تحقق من عدم وجود أخطاء في ملف الأمر.

- **مشاكل في الاتصال بقاعدة البيانات (`MongoDB` / `SQLite`)**
  - **السبب:** خطأ في رابط الاتصال (`mongoURI`) أو مشكلة في خدمة قاعدة البيانات نفسها.
  - **الحل:** تأكد من صحة رابط الاتصال في `config.json`، وتأكد من أن قاعدة البيانات تعمل بشكل سليم.

## ⚠️ ملاحظات هامة وإخلاء مسؤولية
- هذا البوت يعتمد على واجهة برمجة تطبيقات **غير رسمية** لفيسبوك. استخدامها قد يعرض حساب البوت للحظر المؤقت أو الدائم.
- **يُنصح بشدة** استخدام حساب فيسبوك **ثانوي** (Clone) مخصص للبوت فقط لتجنب أي خطر على حسابك الشخصي.
- المطور **غير مسؤول** عن أي ضرر أو حظر قد يحدث لحسابك نتيجة استخدام هذا البوت.
- **لا تقم بتغيير الحقوق** أو إزالة اسم المطور، فأي محاولة لذلك قد تعرضك للحظر من استخدام التحديثات المستقبلية.
